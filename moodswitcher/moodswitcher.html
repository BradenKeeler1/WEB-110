Questions & Answers

Why is the JavaScript textContent property safer than innerHTML?

textContent is safer because it treats everything as plain text, preventing accidental or malicious HTML or JavaScript from being executed. innerHTML interprets input as HTML, which can lead to security risks like cross-site scripting (XSS).

If you use an HTML text box with an attribute like id="text" and a corresponding DOM property such as const box = document.querySelector("#text"), how do they relate?

The HTML id uniquely identifies the element in the document. document.querySelector("#text") retrieves that exact element and stores it in a JavaScript variable so the script can read or modify it.

What would happen to the text box with id="text" if this line were added?

text.textContent = "Hello!" would replace any text inside the element with the word “Hello!”. If the element is an input box, it would not change the typed value because input boxes use the value property instead of textContent.
